## Alpha Hypothesis
- Claim: Public SEC 13F information-table filings, handled with EDGAR acceptance-time discipline and conservative 45-day reporting lag, may identify institutional accumulation/distribution or new/exit positions whose post-publication drift/reversal differs for liquid common stocks.
- Economic / behavioral intuition: Crowded institutional ownership changes disclosed in actual filings may create delayed attention, confirmation, or unwind pressure after public availability; this is a content-based filing signal, not a deterministic filing-calendar effect.
- Expected failure mode: The hard gate is conservative CUSIP-to-ticker/common-stock mapping and breadth. The alpha may collapse into AR-130 filing-calendar attention, generic momentum/reversal, sector/beta, size/liquidity, or stale quarterly disclosure noise.

## Implementation Plan
- Candidate product pool: U.S. liquid common stocks discovered from SEC 13F-HR / 13F-HR/A information-table XML filings after conservative issuer/security mapping; ETFs only as controls, not primary alpha universe.
- Product-selection filters: EDGAR accepted filings only, exclude 13F-NT primary sleeve, exclude options/PUT/CALL and ambiguous/non-common securities, require conservative CUSIP/ticker/common-stock mapping, qfa/Alpaca daily coverage, liquidity, breadth, and concentration gates.
- Selected universe: To be selected ex ante after source/mapping gate; hard reject if <150-250 mapped liquid common-stock issuers or if events are concentrated by filer/issuer/sector/quarter/year.
- Universe-selection rationale / limitations: 13F data are delayed, stale, and often CUSIP-only. The scout must prove a durable parser/mapping/change-table before any performance result is considered.
- Data source: SEC EDGAR submissions/archive/index/info-table XML plus qfa/Alpaca real daily OHLCV; no CSV market data; no daemon; no orders.
- Signal features: Same-filer same-CUSIP quarterly holdings changes, new/exit positions, filer-count accumulation/distribution breadth, value/share changes, all lagged to next tradable session after EDGAR acceptance/public availability.
- Model form: Source/vintage feasibility scout first; only if gates pass, event/ranker fast-falsification using mapped liquid common stocks.
- Parameters / search space: Predeclared change buckets, new/exit buckets, conservative acceptance-time-to-next-session mapping; no performance-ranked source selection.
- Position sizing: If source gate passes, dollar-neutral or beta/sector-aware long/short/ranked sleeve with gross cap and 10 bps primary cost gate; otherwise zero-weight scaffold only.
- Risk constraints: No lookahead to report period date; no unmapped CUSIP inference; no double-counting amendments without explicit policy; reject if calendar/momentum/sector controls dominate or max relevant correlation >0.60.

## Evaluation Plan
- Random-period protocol: Source gate first: prove parser, accession/document paths, quarterly change table, mapping breadth, and concentration safety. If passed, run random event/time windows over qfa/Alpaca real daily bars with costs.
- Metrics: source parser success, mapped issuer/event counts, concentration diagnostics, then median/p25/worst random-window Sharpe, positive-window rate, annualized return/volatility, max drawdown, turnover, hit rate.
- Costs/slippage assumptions: 5/10/20 bps one-way costs; 10 bps primary gate.
- Orthogonality checks vs alpha library: AR-130 filing-calendar attention, generic SEC filing-day control, momentum/reversal, sector/beta, size/liquidity, raw price/volume pressure, shifted acceptance dates, shuffled holdings-change labels.
- Acceptance threshold: Source gate must pass before performance. Performance gate requires post-cost median Sharpe >0, p25 not materially negative, positive-window rate >=55%, 20 bps not collapsed, beats shifted/shuffled/calendar/momentum/sector controls, and max relevant correlation <=0.60.
