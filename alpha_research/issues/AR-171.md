## Implementation Plan
- Candidate product pool: U.S. exchange-listed liquid common stocks with SEC submissions metadata containing 8-K Item 2.02 accepted filings over the qfa/Alpaca overlap; ETFs, funds, ADRs, preferreds, warrants, ambiguous mappings, and non-common securities excluded where identifiable.
- Product-selection filters: SEC `acceptanceDateTime`, accession number and filing detail page reachable; subject issuer CIK maps conservatively to a ticker/common stock with qfa/Alpaca daily coverage; event enters no earlier than the next regular session after EDGAR acceptance; require dropped-event log and issuer/year/month/sector concentration diagnostics.
- Selected universe: To be selected ex ante after source gate. Hard source-gate targets: >=1,000 timestamp-safe Item 2.02 events and >=200 liquid mapped issuers if achievable; otherwise reject as insufficient breadth rather than forcing a performance run.
- Universe-selection rationale / limitations: Current SEC ticker mapping can be survivorship/ticker-change biased. The scout must document mapping caveats and drop ambiguous cases. SEC acceptance time is the event timestamp; do not claim actual earnings-release time unless parsed from a press release and validated.
- Data source: SEC submissions JSON, company ticker mapping, filing index/document pages and likely Exhibit 99 / press-release attachments; qfa/Alpaca real daily OHLCV only if source gate passes; no CSV market data; no daemon; no orders.
- Signal features: Source gate first: accepted Item 2.02 event table, after-hours/regular-hours acceptance tags, attachment/parser success flags, mapped issuer counts, event concentration. If and only if gate passes, predeclared next-session to 1/5-session post-acceptance drift/reversal conditional on timestamp-safe same-day reaction and abnormal volume.
- Model form: Source/parser/breadth feasibility scout first; possible later event-study/ranker fast-falsification only after source gate.
- Parameters / search space: Fixed acceptance-time-to-next-session mapping; fixed 1/5/10-session diagnostic horizons if performance gate becomes eligible; no broad mining of item codes, exhibits, or conditioning variables.
- Position sizing: If source gate passes, dollar-neutral or beta/sector-aware ranked common-stock event sleeve with gross cap and 10 bps primary cost gate; otherwise zero-weight scaffold only.
- Risk constraints: No deterministic earnings-calendar fallback; no trading before EDGAR acceptance; no use of future restatements/amendments; reject if shifted labels, matched non-events, filing calendar, generic momentum/reversal, issuer/year concentration, or price/volume controls dominate; max relevant correlation <=0.60.
