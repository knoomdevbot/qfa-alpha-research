## Implementation Plan
- Candidate product pool: U.S. liquid common-stock issuers with SEC Form 4 filings from 2018-2026, canonical source SEC EDGAR submissions metadata and raw Form 4 XML; OpenInsider only as optional non-authoritative cross-check.
- Product-selection filters: EDGAR acceptance timestamp present, raw ownership XML parseable, issuer CIK/ticker conservatively mapped to liquid qfa/Alpaca common stock, transaction codes and acquired/disposed flags extracted, clustered events deduplicated including amendments.
- Selected universe: To be fixed after source/breadth gate; require at least 500 timestamp-safe mapped clustered imbalance events after filters.
- Universe-selection rationale / limitations: Preflight found SEC company ticker mapping, submissions metadata, acceptanceDateTime, raw Form 4 XML, and non-derivative transaction parsing feasible in a bounded sample. However current ticker mapping has survivorship/ticker-change bias, and prior AR-134 failed because no durable event table was built.
- Data source: SEC EDGAR submissions metadata plus raw Form 4 XML; qfa/Alpaca real daily bars only after event-table gate passes; no CSV-backed market data; no daemon; no orders.
- Signal features: Acceptance timestamp, issuer CIK/ticker, reporting-owner role, officer/director flags, non-derivative transaction code P/S, acquired/disposed flag A/D, transaction date/value/shares known only after acceptance, clustered sale-to-purchase imbalance.
- Model form: Source/parser/mapping feasibility scout first; event-study rule-based alpha only if source gate passes.
- Parameters / search space: Fixed cluster windows, imbalance definitions, event lag from EDGAR acceptance to next liquid session; no optimized thresholds before source gate.
- Position sizing: Event-weighted sleeves with issuer/sector/year/filer caps if performance runs; no live trading.
- Risk constraints: Immediate reject if raw XML parser, path handling, mapping, event breadth, or leakage controls fail.

## Evaluation Plan
- Random-period protocol: First produce timestamp-safe event table. If source gate passes, event-time random windows over 2018-2026 using next-session post-acceptance trading and qfa/Alpaca real bars.
- Metrics: Parser success rate, mapped issuer/event counts, cluster breadth/concentration, Sharpe, event returns, annualized return/vol, max drawdown, turnover, hit rate, median/p25/worst random-window Sharpe.
- Costs/slippage assumptions: 10 bps primary turnover cost; 5 and 20 bps sensitivity if performance evaluation runs.
- Orthogonality checks vs alpha library: Compare against AR-134 Form 4, AR-154 Form 144, AR-130 filing-calendar attention, generic event drift, momentum/reversal, sector/size/liquidity controls, and any existing single-name event streams.
- Controls: Filing-day generic control, transaction-date leakage guard, shifted/shuffled acceptance labels, momentum/reversal, sector/size/liquidity, Form 144/13F overlap, amendment/duplicate exclusion, issuer/year/filer concentration.
- Acceptance threshold: Source gate passes only if parser handles raw XML vs XSL path trap, required fields extract reliably, issuers map conservatively, >=500 timestamp-safe clustered events survive, and concentration gates pass. Performance acceptance requires positive after-cost random-window median, non-hostile p25, controls not dominant, and redundancy gate passes.
