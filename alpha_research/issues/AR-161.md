## Alpha Hypothesis
- Claim: Timestamped director/officer Rule 10b5-1 trading-plan adoption, modification, or termination disclosures in SEC filings may create insider-intent/agency signals distinct from raw Form 4 transaction events; liquid issuer abnormal returns can be tested after a conservative EDGAR acceptance-time lag if a durable parser and issuer mapping succeed.
- Economic / behavioral intuition: Trading-plan changes can reveal future insider selling/buying intent or governance/agency information before realized transactions, potentially distinct from generic SEC filing-calendar attention or Form 4 purchase/sale imbalance.
- Expected failure mode: The disclosures are too stale, sparse, boilerplate-heavy, parser-fragile, dominated by issuer buyback text, or redundant with Form 4/Form 144/generic filing-day effects; source/parser/mapping may fail before performance.

## Implementation Plan
- Candidate product pool: U.S. liquid common-stock issuers with official EDGAR 10-Q/10-K Item 5 / Regulation S-K Item 408 10b5-1 plan-change disclosures from 2023 onward; optional 8-K diagnostics; Form 4 raw XML/remarks only as overlap/control if parser validation succeeds.
- Product-selection filters: SEC company_tickers/submissions mapping, EDGAR acceptance timestamp, conservative next-session entry, common stock only, qfa/Alpaca daily bar coverage, sufficient liquidity/history, exclude issuer repurchase plans, negative/no-event boilerplate, ambiguous tables, duplicates/amendments, and unparseable disclosures.
- Selected universe: To be fixed after source/parser/mapping gate; require at least 150 liquid mapped issuer events after filters with no dominant issuer/person/sector/year/quarter.
- Universe-selection rationale / limitations: The source targets actual 10b5-1 plan-change content, not deterministic filing dates or raw Form 4 transactions. Data begins mainly after recent disclosure rules, so sample length and event breadth may be limited.
- Data source: SEC company_tickers.json, submissions CIK JSON, official EDGAR filing text/HTML/XML primary documents; qfa/Alpaca real daily bars only if source gate passes; no CSV-backed market data; no daemon; no orders.
- Signal features: EDGAR acceptance timestamp, issuer/ticker, filing form/accession/document, plan action type adopted/modified/terminated, action date if disclosed, insider role, shares/value/expiration/scheduled sale window if disclosed, and positive versus negative/no-event classification.
- Model form: Source/parser/mapping feasibility scout first; event-study rule-based alpha only if gate passes.
- Parameters / search space: Predeclared post-acceptance 1/5/20 trading-day diagnostics, separated adoption/modification/termination sleeves, no optimized text thresholds before source gate.
- Position sizing: Event-weighted issuer sleeves with cap by issuer/sector/date, market/sector beta controls if performance runs, no live trading.
- Risk constraints: Hard controls versus AR-134/AR-159 Form 4 source failures, AR-130 filing-calendar attention, AR-145 8-K source scout, AR-154 Form 144, generic momentum/reversal, sector/beta, shifted filing labels, matched non-event issuer filing dates, and insider transaction overlap.
