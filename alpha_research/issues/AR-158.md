## Implementation Plan
- Candidate product pool: All U.S.-listed common stocks and ETFs appearing in timestamp-safe Nasdaq, NYSE, and NYSE American Reg SHO threshold securities archives from 2018-2026, then intersected with liquid qfa/Alpaca tradables.
- Product-selection filters: Timestamp-safe dated source archive, conservative next-session publication lag, liquid common stocks/ETFs only, sufficient qfa/Alpaca daily bars, exclude warrants/preferreds/units/OTC/microcaps and leveraged/inverse products unless separated.
- Selected universe: To be fixed after source/breadth gate; require at least 150 liquid mapped event-symbols and no single-year/meme-window concentration.
- Universe-selection rationale / limitations: Tests official settlement-fail threshold state rather than raw FTD or FINRA volume. Nasdaq and NYSE/NYSE American endpoints were reachable in preflight samples, but bulk probes hit anti-bot/rate-limit behavior and Cboe source was not found.
- Data source: Official exchange Reg SHO threshold-list dated files/APIs plus qfa/Alpaca real daily market bars; no CSV-backed market data; no daemon; no orders.
- Signal features: Add event, active persistence duration, removal event, list-age, pre-event distress/liquidity controls, conservative next-session lag.
- Model form: Source-feasibility scout first; event-study rule-based alpha only if source/breadth gate passes.
- Parameters / search space: Predeclare add/removal/persistence states; fixed next-session and 5/20 day diagnostic windows; no optimized thresholds before source gate.
- Position sizing: Event-weighted long/short or long-only sleeves after mapping; cap issuer/sector/year concentration; no live trading.
- Risk constraints: 10 bps primary costs with 5/20 bps sensitivity if performance gate runs; reject if source, breadth, controls, or cost gates fail.

## Evaluation Plan
- Random-period protocol: First build timestamp-safe source/event table. If gate passes, random event windows over 2018-2026 with conservative next-session trading after list publication.
- Metrics: Source coverage, mapped event count, add/remove/persistence counts, Sharpe, annualized/event return, volatility, max drawdown, turnover, hit rate, median/p25/worst random-window Sharpe.
- Costs/slippage assumptions: 10 bps primary turnover cost; 5 and 20 bps sensitivity.
- Orthogonality checks vs alpha library: Compare against AR-138 SEC FTD, AR-140 FINRA short-volume, generic momentum/reversal, sector/size/liquidity, and existing liquidity-stress/reversal watchlist streams where reconstructable.
- Controls: Shifted list dates, shuffled labels, matched non-threshold distressed names, pre-event momentum/reversal, liquidity, sector/size, meme-period exclusion, and add/remove inversion.
- Acceptance threshold: Source scout passes only if dated source archives are reproducible/timestamp-safe, >=150 liquid mapped events/symbols survive filters, and no concentration failure. Performance acceptance requires positive after-cost random-window median, non-hostile p25, controls not dominant, and max relevant correlation <=0.60.
